# Справочник погрузчиков (Django)

Реализация тестового задания на Django с поддержкой операций по справочнику погрузчиков и их простоям.

## Что реализовано

- CRUD для погрузчиков:
  - поля: `brand`, `number`, `capacity (decimal 3)`, `is_active`, `id`, `modified_at`, `updated_by`;
  - `is_active` по умолчанию = `True`;
  - поиск по номеру погрузчика (вхождение, без учета регистра);
  - сброс фильтра;
  - создание и редактирование через модальное окно на странице справочника;
  - удаление с запретом, если есть зарегистрированные простои.
- CRUD для простоев (инцидентов):
  - `started_at` (обязательное поле, по умолчанию текущее время);
  - `resolved_at` (необязательное);
  - `description`;
  - создание и редактирование через модальное окно на странице справочника;
  - сортировка простоев по `started_at DESC`.
- Расчет времени простоя:
  - `resolved_at - started_at`;
  - если `resolved_at` не задан, берется текущее время;
  - отображение в формате `чч:мм`.
- Подтверждения на операции удаления и отмены несохраненных изменений.
- Интерфейс по макету из ТЗ:
  - красный верхний хедер и кнопка `Профиль`;
  - левое вертикальное меню;
  - двухколоночная зона таблиц (`Погрузчики` и `Простои по погрузчику`);
  - адаптивное поведение для планшетов и мобильных экранов.

## Технологии

- Python 3.13
- Django 6
- PostgreSQL 16 (основная БД)

## Запуск локально

```bash
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
docker-compose up -d
python manage.py migrate
python manage.py runserver
```

Открыть: `http://127.0.0.1:8000/`

## Запуск PostgreSQL

1. Скопируйте `.env.example` в `.env` и при необходимости измените значения.
2. Поднимите БД:

```bash
docker-compose up -d
```

3. Выполните миграции:

```bash
python manage.py migrate
```

4. Остановить БД:

```bash
docker-compose down
```

## Структура

- `config/` — настройки Django и маршрутизация проекта.
- `registry/` — модели, формы, views, urls, миграции для предметной области.
- `templates/registry/` — страницы списка и форм.
- `static/` — стили.
- `docs/conditions.txt` — извлеченный текст исходного условия.

## Публичный деплой (бесплатно) на Render

Проект подготовлен для автодеплоя через `render.yaml`.

1. Загрузите проект в GitHub.
2. На Render создайте новый Blueprint и укажите репозиторий.
3. Render автоматически поднимет:
   - web-сервис Django (`gunicorn`);
   - PostgreSQL database.
4. После первого деплоя получите публичный URL вида `https://<service>.onrender.com`.

Что уже настроено:
- production WSGI запуск через `gunicorn`;
- статика через WhiteNoise;
- автоприменение миграций/collectstatic в `build.sh`;
- подключение к БД через `DATABASE_URL`.
